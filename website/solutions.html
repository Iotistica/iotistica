Zero-Loss Ingestion Pipeline

Agent telemetry arrives over MQTT and lands in a Redis Stream before touching the database. This decouples broker throughput from write latency — bursts are absorbed, workers drain the stream in parallel, and data is committed to TimescaleDB using PostgreSQL COPY for near-native write speeds.

  • Redis Stream buffer: absorbs traffic spikes without backpressure to the broker
  • Parallel workers: configurable pool drains the stream concurrently with no shared locks
  • COPY-based bulk insert: batches bypass row-by-row INSERT for near-disk-speed writes
  • Disk spool fallback: if Redis is unreachable, data spools to disk and replays automatically
  • Ingestion profiles: choose batch, balanced, streaming, or high-performance to match your load

Time-Series Storage & Queries

All readings land in TimescaleDB hypertables — automatically partitioned, compressed after 7 days, and pre-aggregated by 14 continuous aggregate views so your dashboards query summaries instead of raw chunks.

  • 5 hypertables: readings, agent metrics, logs, broker stats, topic metrics
  • 14 continuous aggregates: 1-minute to daily rollups, refreshed in the background
  • LZ4 compression: 85–95% storage reduction on sensor data after 7 days
  • Configurable retention: raw readings kept up to 2 years; aggregates independently managed
  • REST query API: latest values, hourly/daily rollups, and flexible time-range queries

Device Fleet Management

The cloud API maintains a desired state for every device in your fleet. When an agent connects, it receives its target configuration and reconciles — deploying containers, applying settings, or starting protocol adapters — without any manual intervention.

  • Desired-state model: push config to the cloud; agents pull and apply automatically
  • Container orchestration: deploy, update, and remove Docker workloads on any device
  • Jobs: dispatch one-off commands, scripts, or firmware updates to individual devices or fleets
  • Remote access: SSH tunnel and interactive shell to any online device, no port-forwarding needed
  • Provisioning: one-time key flow registers a new device, issues MQTT credentials, and enrolls it in the fleet automatically

Edge Intelligence

Processing at the edge reduces cloud bandwidth, catches anomalies before they reach storage, and keeps devices operational even when connectivity drops.

  • On-device anomaly detection: ML models score each reading locally and flag outliers in real time
  • Protocol discovery: agent scans the local network for Modbus, OPC-UA, and BACnet devices automatically
  • Offline buffer: SQLite queue holds readings during outages; drains to cloud on reconnect with QoS 1
  • Health watchdog: systemd integration and memory leak detection restart the agent if it drifts
  • Multi-architecture: single image runs on ARM64, ARM32, and x86-64 hardware
